🦞 OpenClaw Setup (Step-by-Step)

Follow these steps to connect OpenClaw to Anti-API:

1Install OpenClaw (if not already installed) 
npm install -g openclaw
2Open your config file 
nano ~/.openclaw/openclaw.json

If the file doesn't exist yet, create it. If it already has content, merge the following into your existing config.

3Add the Anti-API provider

Paste this into your openclaw.json. Replace ak-YOUR_KEY with your actual API key:

{
  "models": {
    "mode": "merge",
    "providers": {
      "anti-api": {
        "baseUrl": "https://anti-api-proxy-production.up.railway.app/v1",
        "apiKey": "ak-YOUR_KEY",
        "api": "openai-completions",
        "models": [
          { "id": "claude-opus-4-6-thinking", "name": "Claude Opus 4.6 Thinking", "reasoning": true },
          { "id": "claude-opus-4-5-thinking", "name": "Claude Opus 4.5 Thinking", "reasoning": true },
          { "id": "claude-sonnet-4-6", "name": "Claude Sonnet 4.6" },
          { "id": "claude-sonnet-4-5", "name": "Claude Sonnet 4.5" },
          { "id": "gemini-3-1-pro-high", "name": "Gemini 3.1 Pro High" },
          { "id": "gemini-3-flash", "name": "Gemini 3 Flash" }
        ]
      }
    }
  },
  "agents": {
    "defaults": {
      "model": {
        "primary": "anti-api/claude-opus-4-6-thinking"
      }
    }
  }
}
4Restart the gateway 
openclaw gateway restart
5Verify it works 
openclaw agent --local -m "Hello, what model are you?" --session-id test --json

You should see "provider": "anti-api" in the output.

✅ Tools work natively. OpenClaw's built-in tools (file read/write, web search, process, etc.) work out of the box with all Anti-API models — no extra configuration needed.

🤖 Claude Code Setup (Step-by-Step)

Follow these steps to connect Claude Code to Anti-API:

1Install Claude Code (if not already installed) 
npm install -g @anthropic-ai/claude-code
2Set environment variables

Add these to your shell config (~/.zshrc or ~/.bashrc):

export ANTHROPIC_BASE_URL="https://anti-api-proxy-production.up.railway.app"
export ANTHROPIC_API_KEY="ak-YOUR_KEY"

Then reload your shell:

source ~/.zshrc
3Set your preferred model 
claude config set --global model claude-opus-4-6-thinking

Or use any model from the list above.

4Test it 
claude -p "Hello, what model are you?"

You should receive a response from the model.

5Test with tools 
claude -p "Read the file package.json and tell me the project name"

Claude Code should use the file-read tool and return the project details.

✅ All Claude Code tools work. File editing, bash commands, web search — everything works through the proxy exactly as with a native Anthropic API key.
Note: Do NOT set ANTHROPIC_BASE_URL with a trailing /v1. Claude Code adds its own path. The correct value is the root URL without /v1.

API Reference

API 接口参考

POST /v1/chat/completions

curl https://anti-api-proxy-production.up.railway.app/v1/chat/completions \
  -H "Authorization: Bearer ak-YOUR_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "claude-opus-4-6-thinking",
    "messages": [{"role": "user", "content": "Hello!"}],
    "max_tokens": 1024,
    "stream": false
  }'

Set "stream": true for SSE streaming responses.

设置 "stream": true 可启用 SSE 流式响应。

GET /v1/models

Returns a list of all available models.

返回所有可用模型列表。

Tool Calling

All models support OpenAI-style function/tool calling. The proxy automatically cleans unsupported JSON Schema fields (like patternProperties, $ref, additionalProperties) so your existing tool schemas work without modification.

工具调用

所有模型都支持 OpenAI 风格的函数/工具调用。代理会自动清理不支持的 JSON Schema 字段（如 patternProperties、$ref、additionalProperties），你现有的工具 schema 无需修改即可使用。
